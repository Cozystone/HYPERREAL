## 무엇인가

WETHUS는 README에서 `WETHUS/`를 active static app으로 명시한다. 라이브 사이트는 "모든 학생 창업 프로젝트, 여기서 시작된다"라는 문장을 중심으로 학생 창업, 팀빌딩, 멘토 연결을 한 화면에 조직한다.

## 운영 구조

레포에는 과거 백업 프로젝트가 함께 존재하므로 active path와 reference-only path를 분리해 관리한다. `scripts/validate-static.js`와 GitHub Actions static checks가 정적 배포면의 품질 게이트 역할을 한다.

## 포트폴리오 관점

WETHUS는 학생 창업을 "아이디어 소개"가 아니라 역할, 팀, 지원, 멘토링의 실행 단위로 재배열한다. HYPERREAL에서는 팀 형성 시스템을 만든 프로젝트로 기록한다.
